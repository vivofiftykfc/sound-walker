# 声纹锁三路线对比实验报告

## 一、项目背景与目标

### 1.1 项目目标
在 ARM 嵌入式 Linux 平台实现声纹注册 + 声纹识别功能（离线，非在线服务）。

### 1.2 当前阶段：x86 快速原型对比
在 x86 电脑上对比三条识别路线，选出最佳方案后再移植到 ARM。

### 1.3 三条路线
| 路线 | 方法 | 技术特点 |
|------|------|----------|
| 路线1 | MFCC + DTW | 传统信号处理，可移植 CMSIS-DSP |
| 路线2 | GMM + SVM | 统计学习，sklearn 实现 |
| 路线3 | Deep Embedding | 预训练模型 + 余弦相似度 |

## 三、调试过程

### 3.1 数据集下载问题

**问题**：首次下载数据集时中断，仅获得 3.2MB（不完整），完整包为 322MB。

```
第一次下载: dev-clean.tar.gz = 3.2MB (中断)
正确大小:  dev-clean.tar.gz = 322MB
```

**解决**：
```bash
rm data/dev-clean.tar.gz
wget https://openslr.org/resources/12/dev-clean.tar.gz -O data/dev-clean.tar.gz
tar xzf dev-clean.tar.gz -C data/
```

### 3.2 DTW 验证速度极慢

**问题**：运行脚本后，MFCC+DTW 的验证阶段耗时极长，单次验证达 5.4秒。

```
[MFCC+DTW] Acc=0.808  EER=0.265  Enroll=6.5ms  Verify=5443.1ms
```

**原因分析**：
DTW 算法复杂度为 O(n×m)，其中 n 和 m 分别是两段语音的帧数。一次验证需要：
- 10个说话人 × 8条测试语音 × 10个注册模板 = 800次 DTW 计算
- 每条语音约 200-300 帧，总复杂度约 O(800 × 300²) ≈ 7200万次距离计算

**解决**：将 `MAX_SPEAKERS` 从 20 降至 10，减少计算量。

### 3.3 脚本输出缓冲问题

**问题**：后台运行时 `print()` 输出被缓冲，看不到进度。

**解决**：使用 `python -u` 禁用缓冲，或者 `PYTHONUNBUFFERED=1`。

## 四、算法原理与实现

### 4.1 路线1：MFCC + DTW

#### 原理

**MFCC（梅尔倒谱系数）** 特征提取流程：
1. 预加重（提升高频）
2. 分帧（25ms 窗，10ms 帧移）
3. 加 Hamming 窗
4. FFT 变换到频域
5. 通过 Mel 滤波器组（40个滤波器）
6. 取对数
7. 离散余弦变换（DCT）得到13维 MFCC

**DTW（动态时间规整）**：
- 用于比较两段不等长时序信号的相似度
- 通过动态规划找到最优对齐路径
- 使用 Sakoe-Chiba 约束窗口加速

```
DTW 距离公式：
D[i,j] = dist(i,j) + min(D[i-1,j], D[i,j-1], D[i-1,j-1])
```

#### 实现

```python
def dtw_distance(x: np.ndarray, y: np.ndarray, window: int = None) -> float:
    N, M = len(x), len(y)
    D = np.full((N + 1, M + 1), np.inf)
    D[0, 0] = 0
    dist_mat = cdist(x, y, metric='euclidean')
    for i in range(1, N + 1):
        j_start = max(1, i - window)
        j_end = min(M, i + window) + 1
        for j in range(j_start, j_end):
            cost = dist_mat[i-1, j-1]
            D[i, j] = cost + min(D[i-1, j], D[i, j-1], D[i-1, j-1])
    return D[N, M] / (N + M)  # 归一化
```

#### 声纹验证流程

```
注册阶段：
  语音 → MFCC特征 → 存储为注册模板

验证阶段：
  测试语音 → MFCC特征 → DTW距离(测试, 每个注册模板) → 取最小距离
```

### 4.2 路线2：GMM + SVM

#### 原理

**GMM（高斯混合模型）**：
- 用多个高斯分布的线性组合描述特征的分布
- 每个说话人训练一个独立的 GMM
- 验证时计算对数似然比（LLR）打分

**UBM（通用背景模型）**：
- 用大量说话人数据训练一个全局 GMM
- 作为说话人 GMM 的先验
- 使用 MAP 自适应快速 adapt 到新说话人

**SVM（支持向量机）**：
- 在 GMM supervector 空间训练分类器
- Linear kernel 用于高效分类
- 输出概率用于验证打分

**Supervector 构造**：
```
对于每个高斯分量 k:
  f_k = Σγ_k(x_t)(x_t - μ_k) / √n_k

其中 γ_k 是后验概率，μ_k 是UBM均值，n_k是计数
最终 supervector = [f_1, f_2, ..., f_K] 拼接
```

#### 实现

```python
class GMM_SVM:
    def _supervector(self, mfccs: List[np.ndarray]) -> np.ndarray:
        all_mfcc = np.vstack(mfccs)
        posteriors = self.ubm.predict_proba(all_mfcc)
        sv = []
        for k in range(self.ubm_components):
            gamma = posteriors[:, k]
            n_k = gamma.sum() + 1e-8
            f_k = (gamma[:, None] * (all_mfcc - self.ubm.means_[k])).sum(axis=0) / np.sqrt(n_k)
            sv.append(f_k)
        return np.concatenate(sv)
```

#### 声纹验证流程

```
注册阶段：
  N条语音 → MFCC → 训练GMM → 提取supervector → 训练SVM

验证阶段：
  测试语音 → MFCC → supervector → SVM概率输出 → 识别说话人
```

### 4.3 路线3：Deep Embedding (Resemblyzer)

#### 原理

**Deep Embedding**：
- 使用预训练的 Speaker Verification 模型提取固定维度的 embedding 向量
- Resemblyzer 使用基于 LSTM 的 GE2E (Generalized End-to-End) 损失训练的模型
- 模型输入是 log-Mel 滤波器组输出
- 输出是 256 维或 512 维的说话人嵌入向量

**余弦相似度**：
```
cosine_sim(a, b) = (a · b) / (||a|| × ||b||)
```

#### 实现

```python
class DeepEmbedding:
    def __init__(self):
        from resemblyzer import VoiceEncoder
        self.encoder = VoiceEncoder()

    def enroll(self, spk_id: str, audios: List[np.ndarray]):
        embeds = []
        for audio in audios:
            embed = self.encoder.embed_utterance(audio.astype(np.float32))
            embeds.append(embed)
        self.embeddings[spk_id] = np.mean(embeds, axis=0)

    def verify(self, test_audio: np.ndarray) -> Dict[str, float]:
        test_embed = self.encoder.embed_utterance(test_audio.astype(np.float32))
        results = {}
        for spk_id, enroll_embed in self.embeddings.items():
            sim = np.dot(test_embed, enroll_embed) / (
                np.linalg.norm(test_embed) * np.linalg.norm(enroll_embed) + 1e-8)
            results[spk_id] = sim
        return results
```

#### 声纹验证流程

```
注册阶段：
  N条语音 → VoiceEncoder → 提取embedding → 平均 → 注册模板

验证阶段：
  测试语音 → VoiceEncoder → embedding → 余弦相似度(测试, 每个注册模板) → 识别说话人
```

## 五、实验结果

### 5.1 数值结果

| 指标 | MFCC + DTW | GMM + SVM | Deep Embedding |
|------|-------------|-----------|----------------|
| **准确率** | 80.8% | **96.3%** | 93.8% |
| **EER（等错误率）** | 26.5% | **2.8%** | 5.3% |
| **FAR @ FRR=1%** | 85.5% | 21.0% | 12.7% |
| **注册时间** | 6.5ms | 152.1ms | 233.3ms |
| **验证时间** | 5443.1ms | 7.1ms | 70.6ms |
| **模型大小** | 1005.3KB | 22.8KB | 10.0KB |

### 5.2 结果分析

#### 准确率分析

```
GMM + SVM (96.3%) > Deep Embedding (93.8%) > MFCC + DTW (80.8%)
```

- **GMM + SVM 最高**：利用了 UBM 提供的全局先验信息和 SVM 的强分类能力
- **Deep Embedding 次之**：预训练模型泛化能力强，但可能受限于与训练任务的差异
- **MFCC + DTW 最低**：DTW 只考虑局部相似性，无法捕捉说话人的全局分布特性

#### EER（等错误率）分析

EER 是 FAR = FRR 时的错误率，越低越好：

- **GMM + SVM (2.8%)**：最优，说明正负样本区分度高
- **Deep Embedding (5.3%)**：次优，深度特征表达能力较强
- **MFCC + DTW (26.5%)**：极差，模板匹配方式不适合多说话人场景

#### 速度分析

```
注册速度: MFCC+DTW (6.5ms) > GMM+SVM (152ms) > Deep Embedding (233ms)
验证速度: GMM+SVM (7.1ms) > Deep Embedding (70.6ms) >> MFCC+DTW (5443ms)
```

- **MFCC + DTW 注册快**：只需存储 MFCC 特征，无训练过程
- **MFCC + DTW 验证极慢**：O(n×m) 复杂度，800次DTW计算导致5.4秒/次
- **GMM + SVM 验证快**：只需计算高斯分布的对数似然，矩阵运算高效
- **Deep Embedding 验证中等**：需要神经网络前向传播

#### 模型大小分析

```
Deep Embedding (10KB) < GMM+SVM (22.8KB) < MFCC+DTW (1005KB)
```

- **Deep Embedding 最小**：仅存储256维embedding向量
- **GMM + SVM 次之**：存储GMM参数和SVM权重
- **MFCC + DTW 最大**：需要存储所有注册语音的MFCC特征序列

## 六、各方法优缺点总结

### 6.1 MFCC + DTW

| 优点 | 缺点 |
|------|------|
| 原理简单，易于理解 | 准确率低 (80.8%) |
| 无需训练 | 验证速度极慢 (5.4秒/次) |
| 可完全移植 CMSIS-DSP | 模型模板大 (1MB) |
| 对短语音友好 | 无法捕捉全局分布特性 |

### 6.2 GMM + SVM

| 优点 | 缺点 |
|------|------|
| 准确率最高 (96.3%) | 需要训练过程 |
| EER最低 (2.8%) | 注册速度较慢 (152ms) |
| 验证速度快 (7ms) | 离线可用 |
| 模型小 (23KB) | 可移植 sklearn/cmsisd-dsp |

### 6.3 Deep Embedding (Resemblyzer)

| 优点 | 缺点 |
|------|------|
| 准确率高 (93.8%) | 依赖预训练模型 |
| 模型最小 (10KB) | 验证速度中等 (70ms) |
| 泛化能力强 | 预训练模型需额外存储 |
| 余弦相似度计算简单 | 不完全"离线"（模型来源依赖） |

---

## 七、结论与建议

### 7.1 最佳选择：GMM + SVM

对于 ARM 嵌入式部署，**GMM + SVM** 是最佳选择：

1. **离线可用** — 无需网络连接，不依赖任何在线服务
2. **模型小** — 仅 23KB，易于存储和加载
3. **验证速度快** — 7ms 单次验证，适合实时场景
4. **准确率高** — 96.3% 准确率，EER 仅 2.8%
5. **可移植** — sklearn 可用，C++ 可用移植（GNU GMM/LIBSVM）

### 7.2 备选方案：Deep Embedding

如果能接受预训练模型依赖：

- 模型更小（10KB）
- 准确率也不错（93.8%）
- 但验证速度较慢（70ms）

### 7.3 不推荐：MFCC + DTW

- 准确率太低（80.8%）
- 验证太慢（5443ms）
- 无法接受嵌入式实时需求
