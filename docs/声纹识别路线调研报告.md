| 维度 | 路线一 MFCC+DTW | 路线二 GMM/SVM | 路线三 DL Embedding |

## 三、路线一：MFCC + DTW（传统方法）

### 3.1 处理流程

```
原始PCM音频 → 预加重 → 分帧 → 加窗 → FFT → Mel滤波器组 → Log → DCT → MFCC特征 → DTW匹配
```

### 3.2 各步骤底层原理

#### （1）预加重 (Pre-emphasis)

```
y[n] = x[n] - α·x[n-1]    (α 通常取 0.97)
```

这是一个一阶高通滤波器。语音信号的高频部分（如摩擦音 /s/ /f/）能量天然比低频（元音）低，预加重提升高频、压制低频，使频谱更平坦。

**物理依据：** 声门激励源的频谱以 -12dB/oct 滚降，嘴唇辐射效应再贡献 +6dB/oct，预加重补偿了这 -6dB/oct 的净衰减。

#### （2）分帧 (Framing)

取 20-30ms 为一帧（16kHz 下 = 320-480 个采样点），帧移 10ms（50%-67% 重叠）。

**原理：** 语音在 20-30ms 尺度上近似平稳——发声器官（舌、唇、下颌）的机械运动远慢于声带振动（100-300Hz），满足 FFT 对信号平稳性的要求。

#### （3）加窗 (Windowing)

每帧乘以汉明窗：

```
w[n] = 0.54 - 0.46·cos(2πn/(N-1))
```

**原理：** 直接截断等于乘矩形窗，会导致频谱泄漏——能量从主瓣泄漏到旁瓣，相邻频率分量互相干扰。汉明窗的旁瓣衰减 -43dB（矩形窗仅 -13dB），大幅减少泄漏。

#### （4）FFT (快速傅里叶变换)

将加窗后的时域帧变换到频域，得到幅度谱和相位谱。

**原理：** 人耳对相位不敏感（听觉系统的"相位聋"现象——基底膜不同位置的毛细胞独立响应频率而非相位），所以后续只保留幅度谱 |X(k)|，丢弃相位信息。

#### （5）Mel 滤波器组（MFCC 核心设计）

**人耳的非线性频率感知：** 低频区分辨力强（能区分 100Hz vs 200Hz），高频区分辨力弱（难以区分 8000Hz vs 8100Hz）。Mel 尺度模拟了这种感知特性：

```
mel(f) = 2595 · log₁₀(1 + f/700)
```

在 Mel 尺度上等间距排列 26-40 个三角形滤波器，映射回线性频率后自然变成"低频密集、高频稀疏"的分布。每个滤波器的输出是该频带内频谱能量的加权和。

#### （6）Log 压缩

对每个 Mel 滤波器的能量取对数。

**两个目的：**
- 模拟人耳对响度的对数感知（韦伯-费希纳定律：感知增量与刺激强度的比值成正比）
- 将语音信号中声道滤波（时变卷积）和声门激励的关系从卷积变成加法：`X(f) = G(f)·H(f)` → `log|X| = log|G| + log|H|`，便于后续 DCT 分离

#### （7）DCT (离散余弦变换)

对 Log-Mel 能量序列做 DCT-II，保留前 13-26 个系数：

```
c[i] = Σⱼ log(Eⱼ) · cos(πi(j+0.5)/M)    i = 1,2,...,K
```

跳过 C₀（直流分量，反映总能量，但不稳定）。

**关键作用——去相关：** Mel 滤波器组的输出是高度相关的（相邻滤波器频带有重叠），DCT 近似于 PCA/KLT 变换，将能量压缩到前几个系数，同时使各系数近似独立。去相关后的特征在 DTW 中做欧氏距离计算才合理。

#### （8）DTW (动态时间规整) 匹配

同一个人说同一句话，每次语速不同，帧数不同。假设注册模板有 N 帧 MFCC，测试语音有 M 帧 MFCC，不能直接逐帧做差。

DTW 构建一个 N×M 的累积代价矩阵：

```
D[i,j] = d(i,j) + min( D[i-1, j],    // 测试语音"拉伸"——模板一帧对测试多帧
                       D[i, j-1],    // 测试语音"压缩"——测试一帧对模板多帧
                       D[i-1, j-1] ) // 正常对齐
```

其中 `d(i,j)` 是注册模板第 i 帧和测试语音第 j 帧 MFCC 向量之间的欧氏距离。最终 `D[N, M]` 是两条序列的最小全局对齐代价，与阈值比较完成接受/拒绝判断。

**Sakoe-Chiba 窗约束：** 只允许在对角线附近搜索（`|i - j·N/M| ≤ w`），将计算量从 O(N·M) 降到 O(N·w)，同时防止病态时序弯曲（如一帧对上百帧的极端情况）。

## 四、路线二：GMM + SVM（统计学习）

### 4.1 GMM（高斯混合模型）路线

#### 处理流程

```
MFCC特征 → UBM训练（通用背景模型）→ MAP自适应 → 注册说话人GMM → 对数似然比打分
```

#### 底层原理

**GMM 建模：** 将每个说话人的 MFCC 特征在声学空间中的分布建模为 K 个高斯分量的加权和：

```
P(x|λ) = Σₖ wₖ · N(x|μₖ, Σₖ)
```

其中 `N(x|μ, Σ)` 是多维高斯分布，`wₖ` 是第 k 个分量的混合权重，`μₖ` 是该分量的均值向量（代表声学空间中的一个"典型声音原型"），`Σₖ` 是协方差矩阵（通常简化为对角矩阵以减少参数量）。每个高斯分量捕捉 MFCC 特征空间中一个密集区域——对应一类相似的短时声学模式。

**UBM（通用背景模型）：** 先用大量不同说话人的语音训练一个"通用 GMM"。UBM 在特征空间中密集覆盖了所有可能的语音帧分布，代表"一般人的平均发声特征"。UBM 是说话人自适应的"锚点"。

**MAP（最大后验概率）自适应：** 对于每个注册说话人，用该人的少量语音对 UBM 的各高斯分量进行贝叶斯自适应。核心公式：

```
μ̂ₖ = (τ·μ_ubmₖ + nₖ·x̄ₖ) / (τ + nₖ)
```

其中 `τ` 是"相关性因子"（控制新均值是靠近 UBM 还是说话人样本），`nₖ` 是说话人数据落在第 k 个分量中的"软计数"。当注册数据不足时 μ̂ₖ 倾向于 UBM 的 μ_ubmₖ，保证估计的稳健性。协方差和权重通常保持不变以减少待估参数。

**打分（对数似然比检验）：** 对测试语音的每一帧 MFCC xₜ，分别计算在注册说话人 GMM 和 UBM 下的对数似然，取差值后时间平均：

```
Score = (1/T) Σₜ [log P(xₜ|λ_enrolled) - log P(xₜ|λ_UBM)]
```

这个似然比回答了核心问题：这段语音"更像该特定说话人"还是"更像一般人"？正值越大，越可能是该说话人。与阈值比较完成决策。

### 4.2 SVM 路线

#### 处理流程

```
MFCC特征 → GMM超向量构建 → SVM训练/分类
```

#### 底层原理

**GMM 超向量：** 将 GMM 各分量的均值向量 μₖ 拼接成一个高维向量（如 512 高斯 × 13 维 MFCC = 6656 维）。这个超向量是该说话人的固定维度"指纹"，无论原始语音多长，都被压缩成这个向量。

**SVM 判别：** 在超向量空间中寻找最优分类超平面，最大化正类（目标说话人）和负类（冒名者）之间的间隔。SVM 通过核函数（常用线性核或 RBF 核）将超向量映射到更高维空间以增强线性可分性。

**GMM 与 SVM 的关系：** GMM 是生成式方法（建模 P(X|Speaker)，即"这个说话人会发出什么样的声音"），SVM 是判别式方法（直接建模 P(Speaker|X) 的决策边界，即"这个声音属于谁"）。两者可以融合：GMM 超向量作为 SVM 的输入特征，同时利用 GMM 的特征表达能力和 SVM 的判别能力。在训练样本有限时，判别式方法通常泛化更好。

## 五、路线三：深度学习 Embedding（DNN 声纹嵌入）

### 5.1 处理流程

```
原始PCM → STFT/Mel频谱 → TDNN/ResNet编码器 → Statistics Pooling → Embedding向量 → 余弦相似度匹配
```

### 5.2 各步骤底层原理

#### （1）声学特征输入

不像 MFCC 丢弃相位和做 DCT 去相关，深度学习通常输入 **FBank（Log-Mel 频谱，不做 DCT）**，保留更多原始频谱信息供网络自动学习有用的变换。输入形状为 `[T, F]`（T 帧 × F 个 Mel 频带，典型 F=80）。

DNN 的强大之处在于：它可以自动学习比 DCT 更优的去相关和特征压缩变换，不需要人工设计。

#### （2）TDNN（时延神经网络）编码——x-vector / ECAPA-TDNN 的核心构件

TDNN 本质是一维卷积在时间轴上的滑动，每个卷积核覆盖当前帧 ± 前后若干帧的上下文：

- **底层 TDNN：** 感受野小（±2 帧），捕捉短时声学细节——音素级别的频谱模式
- **中层 TDNN：** 感受野扩大，捕捉音素序列的过渡模式
- **高层 TDNN：** 感受野覆盖整句，捕捉全局韵律特征——语调曲线、说话节奏、嗓音特质

**通俗类比：** 底层"看"单个音素 /a/，中层"看"音素序列 /h-e-l-ou/，高层"看"整句 "Hello" 的语调曲线和发声风格。

**ECAPA-TDNN 的改进：** 
- 引入 **SE-Res2Block**（Squeeze-Excitation Res2Net 模块），通过通道注意力机制自动学习哪些频带对说话人区分更重要
- 多层特征融合，综合利用不同粒度的声学特征

#### （3）Statistics Pooling（统计池化）——时间维度压缩

网络在时间维度的输出是 `[T, D]`（T 帧 × D 维特征）。需要将变长序列压缩为固定长度的说话人表征向量。三种策略：

- **Temporal Average Pooling (TAP)：** 对所有帧取平均——简单但假设每帧同等重要，无法区分"话多的帧"和"沉默的帧"
- **Self-Attentive Pooling (SAP)：** 网络自动学习每帧的注意力权重 αₜ，加权求和——语音信息密集的帧（如元音段）自动获得更高权重
- **Attentive Statistics Pooling (ASP)：** 同时输出加权均值 μ 和加权标准差 σ，捕获帧级别分布的二阶统计量。ECAPA-TDNN 使用此方法，因为说话人特征不仅体现在"平均频谱"（一阶），也体现在"频谱的波动模式"（二阶）

#### （4）Embedding + AAM-Softmax 训练

Pooling 后的向量通过全连接层映射为 192 维（或 256/512 维）的说话人嵌入向量（speaker embedding），并做 L2 归一化，将所有说话人的 embedding 约束在单位超球面上。

**AAM-Softmax（加性角度边缘 Softmax）损失函数：**

```
Loss = -log( exp(s·cos(θ_y + m)) / [exp(s·cos(θ_y+m)) + Σ_{j≠y} exp(s·cos(θⱼ))] )
```

其中：
- `θ_y` 是当前 embedding 与"正确说话人"类别中心向量的夹角
- `m` 是手动添加的角度惩罚项（典型值 0.2-0.3 弧度）
- `s` 是缩放因子（典型值 30-64）

**直观理解：** 普通 Softmax 只要求 embedding 离正确类别中心"近一点"。AAM-Softmax 要求它离正确中心"近一整个角度余量 m"，同时远离所有错误类别中心。这迫使优化后的 embedding 在超球面上形成紧密的类内聚簇和宽大的类间间距，是声纹嵌入取得高区分度的关键技术。

#### （5）声纹匹配与决策

注册时存储 embedding 向量 e_enroll，识别时计算测试 embedding e_test：

```
cos_sim = (e_enroll · e_test) / (||e_enroll|| · ||e_test||)
```

由于 L2 归一化后 `||e|| = 1`，余弦相似度退化为点积，计算量仅为 O(D)，即常数时间（与语音长度无关，因为 pooling 已经压缩为固定维度）。与阈值比较完成决策——阈值可通过验证集调优，平衡误接受率 (FAR) 和误拒绝率 (FRR)。

## 六、推荐开发路径

### 第一阶段（快速出 MVP）：路线一 MFCC + DTW

**理由：**
- 与参考文献 [1] 高度对应，学术报告有据可依
- CMSIS-DSP 提供完整的 MFCC + DTW API，单一可信赖依赖
- STM32 → ARM Linux 移植路径清晰
- RAM < 200KB，CPU 余量充裕
- 快速出可运行原型，建立信心

**技术栈：**
- ARM CMSIS-DSP（MFCC + DTW）
- dpirch/libfvad（端点检测）
- xiph/speexdsp（降噪）
- ALSA（音频采集）

### 第二阶段（提升准确率）：路线三 sherpa-onnx + ECAPA-TDNNLite

**理由：**
- 准确率从 ~90% 提升到 ~97%
- sherpa-onnx 提供开箱即用的 C++ API
- 有丰富的 benchmark 数据可写入学术报告
- 模型可直接拿来用，不需自己训练
